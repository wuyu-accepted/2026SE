# 服务器部署方案

## 一、架构概览

本项目采用**双服务器**部署架构：

```
┌──────────────────────────────┐      ┌──────────────────────────────┐
│       应用服务器 (App)        │      │      数据库服务器 (DB)        │
│                              │      │                              │
│  Nginx (80端口)              │      │  PostgreSQL / Kingbase       │
│  ├─ 托管 Vue 辅导员端静态文件  │      │  (5432端口)                  │
│  ├─ 反向代理 /api -> 后端     │      │                              │
│  └─ 支持小程序访问            │      │  仅对内网开放，防火墙限制      │
│                              │      │                              │
│  Spring Boot 后端 (18080端口) │      │                              │
│  ├─ Java 17 + Spring Boot 3  │      │                              │
│  ├─ Flyway 自动建表/迁移      │◄────►│                              │
│  └─ 文件上传存储 (本地磁盘)    │      │                              │
└──────────────────────────────┘      └──────────────────────────────┘
```

### 组件说明

| 组件 | 部署位置 | 端口 | 说明 |
|---|---|---|---|
| Nginx | 应用服务器 | 80 | 托管前端静态资源 + 反向代理 API |
| Spring Boot 后端 | 应用服务器 | 18080 | Java 进程，提供 REST API |
| PostgreSQL | 数据库服务器 | 5432 | 业务数据存储 |
| 微信小程序 | 用户手机 | — | 直接请求应用服务器公网 IP/域名 |
| 辅导员 Vue 端 | 浏览器 | — | 通过浏览器访问应用服务器公网 IP/域名 |

## 二、前置条件

### 2.1 服务器要求

| 项目 | 应用服务器 | 数据库服务器 |
|---|---|---|
| 操作系统 | Ubuntu 22.04 / CentOS 7+ | Ubuntu 22.04 / CentOS 7+ |
| 最低配置 | 2核4G | 2核4G |
| 推荐配置 | 4核8G | 4核8G |
| 公网 IP | 是 | 否（内网互通即可） |
| 开放端口 | 80 (HTTP)、18080 (后端 API) | 5432 (数据库，仅对应用服务器开放) |

### 2.2 软件要求

**应用服务器：**
- JDK 17+
- Maven 3.9+（或用项目自带的 `mvnw`）
- Node.js 18+ 和 npm
- Nginx

**数据库服务器：**
- PostgreSQL 16（推荐）或 KingbaseES V8

### 2.3 网络要求

- 应用服务器和数据库服务器之间内网互通
- 数据库服务器防火墙放行应用服务器的内网 IP 访问 5432 端口

## 三、数据库服务器部署

### 3.1 安装 PostgreSQL

```bash
# Ubuntu
sudo apt update
sudo apt install -y postgresql-16

# 启动并设为开机自启
sudo systemctl enable postgresql
sudo systemctl start postgresql
```

### 3.2 初始化数据库

使用项目提供的初始化脚本：

```bash
# 将项目中的脚本复制到数据库服务器
# 或用 scp 从本地上传
scp deploy/db-server/init.sql user@db-server:/tmp/

# 执行初始化
sudo -u postgres psql -f /tmp/init.sql
```

脚本会创建：
- 数据库 `ruc_platform`
- 可选：专用用户 `ruc_admin`

### 3.3 配置远程访问

编辑 `postgresql.conf`：

```bash
sudo vi /etc/postgresql/16/main/postgresql.conf
```

找到并修改：

```
listen_addresses = '*'          # 监听所有网络接口
```

编辑 `pg_hba.conf`：

```bash
sudo vi /etc/postgresql/16/main/pg_hba.conf
```

添加（在文件末尾）：

```
# 允许应用服务器内网 IP 访问
host    ruc_platform    postgres    应用服务器内网IP/32    md5
# 或整段内网网段
host    ruc_platform    postgres    192.168.0.0/16        md5
```

重启 PostgreSQL：

```bash
sudo systemctl restart postgresql
```

### 3.4 防火墙配置

```bash
# 仅允许应用服务器访问
sudo ufw allow from 应用服务器内网IP to any port 5432
sudo ufw enable
```

## 四、应用服务器部署

### 4.1 一键部署（推荐）

在项目根目录执行部署脚本：

```bash
# 先修改 deploy/app-server/deploy.sh 中的数据库连接信息
# DB_HOST、DB_PORT、DB_NAME、DB_USER、DB_PASSWORD

# 赋予执行权限
chmod +x deploy/app-server/deploy.sh

# 执行部署
sudo bash deploy/app-server/deploy.sh
```

脚本会自动完成：
1. 构建 Vue 前端
2. 构建 Spring Boot 后端
3. 配置 Nginx
4. 创建 Systemd 服务
5. 启动后端服务

### 4.2 手动部署步骤

如果需要在部署脚本基础上进行定制，也可以按以下步骤手动部署。

#### 4.2.1 构建 Vue 前端

```bash
cd web-counselor
npm install
npm run build
# 产出在 web-counselor/dist/ 目录
```

#### 4.2.2 构建 Spring Boot 后端

```bash
cd demo
chmod +x mvnw
./mvnw -B -DskipTests clean package
# 产出在 demo/target/ruc-service-platform-1.0.0-SNAPSHOT.jar
```

#### 4.2.3 配置 Nginx

复制 Nginx 配置文件：

```bash
sudo cp deploy/nginx.conf /etc/nginx/conf.d/ruc-platform.conf
# 根据实际情况修改 server_name 和 root 路径
sudo nginx -t        # 检查配置
sudo systemctl reload nginx  # 重载
```

#### 4.2.4 启动后端

使用 `application-server` profile 启动（连接远程数据库）：

```bash
# 直接启动
java -jar demo/target/ruc-service-platform-1.0.0-SNAPSHOT.jar \
    --spring.profiles.active=server \
    --server.port=18080 \
    --DB_HOST=数据库服务器IP \
    --DB_PORT=5432 \
    --DB_NAME=ruc_platform \
    --DB_USER=postgres \
    --DB_PASSWORD=your_password

# 或使用 Systemd 管理（推荐）
sudo cp deploy/app-server/ruc-platform.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable ruc-platform
sudo systemctl start ruc-platform
```

查看后端日志：

```bash
sudo journalctl -u ruc-platform -f
```

## 五、配置文件说明

### 5.1 后端 Profile 体系

| Profile | 配置文件 | 用途 | 数据库位置 |
|---|---|---|---|
| `server` | `application-server.yml` | **应用服务器部署**（推荐） | 远程数据库服务器 |
| `prod` | `application-prod.yml` | 单机生产部署 | 本机 Kingbase |
| `docker-postgres` | `application-docker-postgres.yml` | Docker 部署 | Docker 内 PostgreSQL |
| `default` | `application.yml` + 各环境配置 | 本地开发 | H2 / 本地数据库 |

**双服务器部署必须使用 `server` profile。**

### 5.2 Nginx 配置

`deploy/nginx.conf` 完成以下功能：
- 托管 Vue 辅导员端静态文件
- 反向代理 `/api/*` 到后端 `http://127.0.0.1:18080`
- 支持 SPA 路由历史模式（`try_files` 回退到 `index.html`）
- Gzip 压缩
- 静态资源缓存
- 大文件上传（20MB 限制）
- CORS 已在后端处理，Nginx 无需额外配置

### 5.3 Systemd 服务

`deploy/app-server/ruc-platform.service` 实现：
- 后端开机自启
- 崩溃自动重启（Restart=always）
- 标准日志输出，可通过 `journalctl` 查看

## 六、前端配置

### 6.1 辅导员 Vue 端

Vue 端通过构建时环境变量 `VITE_API_BASE_URL` 配置 API 地址：
- **生产构建时**：不设此值（默认 `''`），API 通过 Nginx 同源反向代理访问
- 如需独立域名，构建时设置：
  ```bash
  VITE_API_BASE_URL=https://api.yourdomain.com npm run build
  ```

### 6.2 微信小程序

修改 `miniprogram-1/utils/config.js` 中的 `BASE_URL`：

```js
const BASE_URL = 'https://你的应用服务器域名或公网IP'
```

小程序端需在微信开发者工具中：
1. 关闭域名校验（开发调试用）
2. 或在小程序后台配置 `request` 合法域名

## 七、验证部署

### 7.1 检查后端

```bash
curl http://localhost:18080/api/home
# 应返回 JSON 响应，code = 0

curl http://localhost:18080/api/auth/me
# 应返回 401 未登录（说明 API 正常工作）
```

### 7.2 检查前端

在浏览器中访问应用服务器公网 IP（或域名）：
- 应能正常打开辅导员登录页面
- `/api/*` 请求应正常代理到后端

### 7.3 检查小程序

在微信开发者工具中配置 `BASE_URL` 后：
- 登录流程应正常
- 首页数据应正常加载
- 党团进度、知识库等功能应正常

## 八、常见问题

### 8.1 后端启动失败：数据库连接超时

- 检查数据库服务器防火墙是否放行了 5432 端口
- 检查 `pg_hba.conf` 是否配置了应用服务器 IP 的访问权限
- 检查数据库服务器上的 PostgreSQL 是否正在运行

### 8.2 后端启动失败：Flyway 校验失败

- 清理脏数据后重建：
  ```bash
  # 在数据库中
  DROP SCHEMA public CASCADE;
  CREATE SCHEMA public;
  ```
- 或设置 `spring.flyway.clean-disabled: false` 临时启用清理（生产环境谨慎操作）

### 8.3 前端页面白屏 / 路由 404

- 检查 Nginx `try_files $uri $uri/ /index.html;` 配置是否正确
- 检查 `root` 路径是否指向 Vue 构建的 `dist` 目录

### 8.4 CORS 错误

- 后端已通过 `CorsConfig.java` 全局允许跨域
- 如果使用 Nginx 反向代理，确保 `/api/` 路径正确代理到后端

### 8.5 文件上传失败

- 确保后端上传目录存在并可写：
  ```bash
  sudo mkdir -p /opt/ruc-platform/uploads
  sudo chown -R www-data:www-data /opt/ruc-platform/uploads
  ```
- 检查 Nginx `client_max_body_size` 是否足够

```
deploy/

├── app-server/
```

## 十、Docker 替代方案（单机部署参考）

如果将来想用 Docker 在应用服务器上运行后端，参考 `docker-compose.postgres.yml`：

```bash
# 仅启动后端容器（数据库已独立部署）
docker run -d \
  --name ruc-platform-app \
  -p 18080:18080 \
  -e SPRING_PROFILES_ACTIVE=server \
  -e DB_HOST=数据库服务器IP \
  -e DB_PORT=5432 \
  -e DB_NAME=ruc_platform \
  -e DB_USER=postgres \
  -e DB_PASSWORD=your_password \
  -v /opt/ruc-platform/uploads:/app/uploads \
  ruc-platform-app:latest
```
