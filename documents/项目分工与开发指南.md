## 二、分工说明

本项目按**功能板块**划分，每个人**独立负责一个板块的前端 + 后端**，互不依赖。

> 注意：各板块需调用的接口已在后端预先实现或预留，分工同学只需在自己负责的板块内完成前后端联调。如遇接口不满足需求，可自行在后端对应模块中新增或修改。

### 2.1 已就绪的基础能力（无需修改）

以下内容已经完成，各个板块基于它们工作即可：

| 内容 | 说明 |
|---|---|
| 后端框架 | Spring Boot 3 + MyBatis-Plus + Sa-Token + Flyway |
| 用户认证 | 注册、登录、微信登录、`/api/auth/**` |
| 审批全链路 | 学生端提交请假 → 管理端审批（Vue 端已完成） |
| 首页聚合 | `/api/home` 提供首页数据 |
| 管理端框架 | 侧边栏导航、路由、工作台页面均已就绪 |

## 三、本地开发指南

### 3.1 启动后端

```bash
cd demo
mvnw.cmd spring-boot:run
```

默认端口 18080，H2 内存库免安装。

或使用 Docker 统一环境：

```bash
docker compose -f docker-compose.postgres.yml up -d --build
```

### 3.2 启动管理端

```bash
cd web-counselor
npm install
npm run dev
```

默认地址 `http://localhost:5173`，首次访问 `/login` 进入。

### 3.3 启动小程序

微信开发者工具导入 `miniprogram-1/`，关闭域名校验，配置 `BASE_URL` 指向后端地址。

## 四、开发规范

### 4.1 Git 提交格式

```
feat(notice): 管理端通知发布页面
feat(party): 学生端党团进度联调
fix(profile): 个人资料保存报错
```

### 4.2 管理端新增页面

1. 在 `web-counselor/src/views/` 下新建 Vue 文件
2. 在 `web-counselor/src/router/index.js` 中将对应路由的 `component` 从 `PlaceholderView` 替换为你的组件
3. 如需新增 API 调用，在 `web-counselor/src/api/` 下新建文件或追加到已有文件

### 4.3 后端新增接口

1. 在对应的包下新建 Controller → Service → Mapper
2. 管理端接口建议路径前缀为 `/api/admin/`
3. 新增接口后同步更新 `documents/API.md`

## 五、验收标准

- 通知板块：学生端可查看通知列表 → 管理端可发布通知 → 学生端看到新通知
- 党团板块：学生端可查看进度和阶段 → 管理端可推进阶段 → 学生端状态更新
- 个人中心：学生端可查看/编辑资料 → 注册流程可跑通（最后做）
