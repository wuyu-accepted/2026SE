# 双端登录调试说明

> 更新时间：2026-05-17  
> 适用对象：项目全体成员

---

## 一、概述

本项目包含两个需要登录的客户端：

| 端 | 技术栈 | 入口 | 适用用户 |
|---|---|---|---|
| **学生端（小程序）** | 微信小程序 | `miniprogram-1/` | 学生 |
| **管理端（Vue 网页）** | Vue 3 + Element Plus | `http://localhost:5173` | 辅导员、管理员 |

两个端共用同一套后端登录接口（`/api/auth/login`），通过角色区分权限。

## 三、启动步骤

### 3.1 一键启动（推荐，需要 Docker）

在项目根目录执行：

```bash
docker compose -f docker-compose.postgres.yml up -d --build
```

启动后：
- 后端 API：`http://localhost:18080`
- 数据库：PostgreSQL `localhost:5432`

二次登录用：
```bash
docker compose -f docker-compose.postgres.yml up -d --build app
```

### 3.2 分步启动（不需要 Docker）

```bash
# 终端 1：启动后端
cd demo
mvnw.cmd spring-boot:run

# 终端 2：启动管理端 Vue
cd web-counselor
npm install
npm run dev
```

管理端默认地址 `http://localhost:5173`。

### 3.3 启动小程序

1. 打开微信开发者工具
2. 导入 `miniprogram-1/` 目录
3. 在"详情 → 本地设置"中勾选"不校验合法域名"
4. 确认 `miniprogram-1/utils/config.js` 中的 `BASE_URL` 指向后端地址（默认 `http://127.0.0.1:18080`）
5. 点击"编译"

## 四、登录流程

### 4.1 小程序端

1. 打开小程序，进入登录页
2. 输入学号和密码（或用上面的测试账号）
3. 点击"登录"
4. 登录成功后自动跳转首页

### 4.2 管理端

1. 浏览器访问 `http://localhost:5173/login`
2. 输入学号和密码
3. 点击"登录"
4. 登录成功后跳转工作台

管理端登录页提供"管理员"和"辅导员"快捷填入按钮，方便调试。

## 五、常见登录问题排查

### 5.1 前端报 `ECONNREFUSED 127.0.0.1:18080`

后端没有启动。检查：
- Docker 容器是否运行：`docker ps`
- 或本地 `mvnw.cmd spring-boot:run` 是否正常启动
- 日志是否有 Flyway 报错

### 5.2 后端报 `FlywayValidateException`

数据库迁移校验失败。通常是某个迁移 SQL 文件被修改过，与之前运行过的版本不一致。

解决方法：重置数据库卷后重建
```bash
docker compose -f docker-compose.postgres.yml down -v
docker compose -f docker-compose.postgres.yml up -d --build
```

### 5.3 登录提示"学号或密码错误"

- 确认密码是否正确（注意大小写）
- 确认数据库中该用户的 `password_hash` 是否已被正确初始化
- Docker 环境下 `AdminAccountInitializer` 会在启动时自动更新密码，等待几秒再试

### 5.4 小程序登录后白屏或接口报错

- 确认 `BASE_URL` 指向正确的后端地址
- 确认小程序开发者工具已关闭域名校验
- 确认后端已启动且可以访问

### 5.5 管理端登录后看不到"系统设置"

- 确认使用的是管理员账号（`admin`/`admin123`）而不是辅导员账号
- 确认登录时 `currentUser` 已正确保存到 localStorage
- 可在浏览器开发者工具 Application → Local Storage 中查看 `currentUser` 的 `roles` 字段

## 六、数据库初始化说明

首次启动时，Flyway 自动执行 `demo/src/main/resources/db/migration/` 下的 SQL 迁移脚本：

- **V1~V13**：建表 + 种子数据（角色、学生、知识库、党团阶段等）
- **V14**：创建管理员和辅导员测试账号（密码初始为 `PLACEHOLDER`）

应用启动后，`AdminAccountInitializer.java` 会自动将密码更新为正确的 BCrypt 哈希：

| 学号 | 初始化密码 |
|---|---|
| `admin` | `admin123` |
| `counselor` | `counselor123` |
| `2023001` | `password` |
| `2023002` | `password` |
| `2023003` | `password` |

如果新增测试账号，在 `AdminAccountInitializer.java` 的 `seeds` 列表中添加即可。
