AI 助手实现架构可视化

展示学生端 AI 助手、管理端模型配置、后端编排服务、知识库检索、功能入口推荐和 DeepSeek/OpenAI 兼容模型调用之间的关系。核心原则：供应商密钥只在后端，用户权限决定可见知识与入口，模型不可用时自动降级。

RUC Platform · AI Assistant

学生端入口

Mini Program 首页左滑 / 悬浮按钮 进入 pages/ai-chat/ai-chat，底栏仍保持 4 个主入口。

对话页 用户输入问题，发送到平台后端 POST /api/ai/chat，页面展示回答、引用知识和可跳转功能入口。

后端 AI 编排核心

Spring Boot AiChatService
  • 校验问题长度与登录用户
  • 检索知识库上下文
  • 按角色匹配功能入口
  • 读取当前 AI 供应商配置
  • 调用模型或降级回答
  • 保存会话与消息审计

管理端配置

Admin 系统设置 / AI 助手配置 管理员维护 DeepSeek、OpenAI 兼容供应商、模型、参数、API Key、检索数量、提示词，并可设为当前或取消当前。

密钥边界 API Key 经 AES-GCM 存储在后端，前端只显示脱敏值，不暴露供应商密钥。

一次学生提问的调用链

1小程序 AI 页用户输入自然语言问题，携带登录 token 请求后端。
→
2权限与上下文Sa-Token 校验；按学生/干部角色过滤可见能力。
→
3RAG-lite知识库检索可见资料，功能入口服务匹配可访问页面。
4模型配置读取当前 active 且 enabled 的供应商配置。
→
5DeepSeek / 兼容 API通过 OpenAI-compatible Chat Completions 调用模型。
→
6统一响应返回 answer、citations、actions、fallback，并保存消息记录。

管理端配置生命周期

新增/编辑管理员填写 provider、baseUrl、model、API Key 和参数。
→
密钥处理后端保存 api_key_cipher，前端只展示 api_key_mask。
设为当前/取消同一按钮切换 active；取消后学生端进入降级模式。
→
测试连接后端用测试 prompt 调用供应商，返回成功、错误和耗时。

降级与安全边界

当没有当前配置、API Key 缺失、供应商调用失败或超时时，AI 助手不会直接失败；后端返回基于知识库和功能入口的规则化建议，并标记 fallback=true。
学生端只访问平台后端 后端负责权限过滤 管理员控制模型供应商 外部模型不可直接接触用户端密钥
